#T-Bank — Blocking Payments Case#
В сервисе по отправке платежей юридическим лицам периодически возникают ситуации, когда платежи надо приостановить, потому что клиент ненадежен (есть подозрение на мошенничество и нужно время разобраться), либо клиент предоставил не те реквизиты для перечисления и платежи отбиваются банком клиента (нужно время на выяснение верных реквизитов). 

##Запроса бизнеса##:
1) Разработать спецификацию в формате OpenAPI endpoint’ов для реализации возможности:
заблокировать платежи конкретного клиента 
разблокировать платежи клиента
роверить, заблокирован ли клиент
тличать блокировки мошенников от блокировок добропорядочных клиентов
2) Разработать структуру хранения необходимой для 1 пункта информации в БД

##Обзор спецификации API##
./openapi.yaml

POST /clients/{clientId}/blocks
Назначение: Завести активную блокировку клиенту.

GET /clients/{clientId}/blocks/active
Назначение: Проверка — заблокирован ли клиент и по какой причине.

POST /clients/{clientId}/blocks/release
Назначение: перевести текущую активную блокировку в RELEASED.
